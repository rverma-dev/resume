## Learned User Preferences

- For this resume workspace, prefer compact, PDF-friendly resume layout that avoids unnecessary blank lines and makes strong use of page width while remaining ATS-friendly.
- Prefer simple resume aesthetics over decorative dividers; use subtle styling only when it clearly improves readability.
- The resume top section should stay headingless: name/contact, two terse identity/proof lines, then Experience; avoid long repeated keyword lines.
- Prefer scanable impact-first resume bullets: split dense technical primitive lists, reduce competing bold text, and reserve bold emphasis for the strongest metrics.
